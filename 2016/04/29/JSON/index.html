JSON

发表于 2016-04-29   |   分类于 JavaScript

三种类型

简单值

可以表示字符串、数值、布尔值和null，但不支持undefined
表示字符串必须用双引号

对象

  与javascript中稍有不同
  javascript中：
      var person = {
           name:"Nicholas",
           age:29
         };
  JSON中：
      {
           "name":"Nicholas",
           "age":29
      }    
三点不同：
  1.没有声明变量（没有变量概念） 
  2.没有末尾分号
  3.属性要求加双引号（其实javascript也可以加）

数组

同样没有声明和分号
[24,"name","age"];

解析与序列化

XML要解析成DOM文档再从中提取数据，极为麻烦
JSON可以解析为javascript对象
JSON变成web服务开发中交换数据的事实标准

JSON对象

ECMAScript 5 定义了一个原生的JSON对象，可以调用 stringify()和parse()，
IE8+，ff3.5+，safari4+，oprea10.5，chrome：JSON解析使用eval()函数
低版本：使用一个shim

stringify() //把javascript对象序列化为JSON字符串

   var person = {
     name:"Nicholas",
     age:29
   };
var jsontext ＝ JSON.stringify(person);
序列化时会忽略函数以及原型成员（prototype），undefined也会跳过，只返回有效的实例属性
序列化选项：上述函数还可以接受两个参数，过滤器和是否保留缩进
 过滤器如果是一个数组，那么只返回该数组中的属性
 如果是一个函数，则可以通过函数对属性操作
 var jsontext = JSON.stringify(person,function(key,value){
     switch(key){
         case "name":return undefined(相当于跳过属性)
         case "age":return 50(更改属性)
         case "一个数组名"：return value.join(",")(把数组连接成字符串)
         default:
                 return value;  保证其他属性都正常返回
})
字符串缩进：var jsontext = JSON.stringify(person,null,4)
缩进最大10个字符
如果缩进参数为字符串，则被用作缩进字符
建议用"- -"或制表符

toJSON()方法

写在对象中，作为一个属性：toJSON:function(){return this.tite;}//返回标题
还可以返回undefined，如果是子对象，会导致返回null
序列化对象的顺序：
1.存在toJSON且能获得有效值，则调用，否则返回对象本身
2. 如果提供函数过滤器，传入过滤器是第一步返回的值。
3.序列化
4.有第三个参数，格式化

parse() // 把JSON字符串变成javascript值

var personCopy ＝ JSON.parse(jsontext);
如果字符串不是有效JSON，会报错。
解析选项
接收第二个函数参数，在每个键值对上调用
返回undefined，则表示删除该属性
releaseDate： new Date(2011,11,1) // 在对象里新增一个属性
key == "releaseDate" // 解析到这个属性
return new Date(value);     //返回一个日期对象为releaseDate属性的值
alert(personCopy.releaseDate.getFullYear()); //这样实现了一个日期
