块级语句

没有作用域，for（定义var）和外面定义一样

var语句：

var a ＝ b ＝ 1；
b会变成全局变量

try－catch语句：

异常捕获
 try{
     throw"test";
}catch(ex){
     console.log(ex); //test
}finally{
     console.log('finally');     
}                     
     有异常执行catch，怎样都会执行finally。

function语句：

function fd(){
...
}  //函数声明语句

var fe = function(){

}; //函数表达式语句，有分号。

函数声明会被预处理，所以在最前面执行fd(); //true
fe() ;//false

for … in语句:

1.顺序不确定
2.enumerable为false不会出现
3.for in 对象属性受原型链影响

with语句：

让JS引擎优化更难
可读性差
可被变量定义代替
严格模式下禁用
